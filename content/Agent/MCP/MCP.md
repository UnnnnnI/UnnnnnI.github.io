
LLM의 한계를 극복하기 위한 기술 -> 대표적으로 파인튜닝 / 프롬프팅 / RAG

파인튜닝: LLM을 특정 도메인이나 작업에 맞게 추가 학습시켜 모델을 조정하는 방법
근데 많은 양의 도메인 특화 데이터가 필요하고, 컴퓨팅 자원과 시간이 필요함

프롬프팅: 효과적인 지시문으로 원하는 작업을 수행하도록 유도

RAG: 외부 데이터베이스/문서를 연결하여 최신 정보를 제공
근데 LLM이 데이터 검색만 가능하고 직접적인 시스템 조작은 불가능했음

직접 시스템 조작하게 하려면 연결 작업을 수행해줘야 함 (미리 정의한 함수 리스트와 매핑하여 실행하는 작업 등)

-> 그래서 Function calling / Tool use 같은 개념이 나오기 시작함
근데 이것도 AI 업체마다 서로 다른 구현 방식을 사용해서 플랫폼 별 다른 코드 작성이 필요했음

-> 그래서 이걸 통합하기 위한 표준화된 프로토콜을 만들고자함
그게 MCP (Model Context Protocol)
LLM에 내가 만든 함수 또는 남이 만든 함수를 연결하는 표준
Tools / Resources (파일, 데이터베이스, API 등 다양한 데이터 소스에 접근하기 위한 통일 방법) / Prompts (재사용 가능한 프롬프트 템플릿을 공유, 관리)
