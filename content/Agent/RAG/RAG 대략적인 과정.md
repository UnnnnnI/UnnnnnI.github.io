
RAG의 원리는 검색 - 정보 증강 - 응답 생성으로 구성됨

1 검색: 사용자 질문에 가장 관련성이 높은 정보를 검색해서 LLM에 전달
([[벡터 검색 (Vector Search)]]기술을 활용하면 데이터베이스에서 질문과 유사한 문서를 효율적으로 찾아 LLM에 제공 가능)

2 정보 증강: 검색된 문서를 원래 입력에 추가해서 모델에게 제공하는 단계

3 응답 생성: LLM이 최종적인 응답을 생성하는 단계


따라서, 전체적인 구현 과정은
"DB 구축 → 벡터 검색 → LLM과 통합" 으로 이루어짐

DB는 벡터 형태로 체계적이고 구조화된 방식으로 저장
검색은 사용자 질문을 벡터 형태로 변환하여 검색




==================================================
RAG의 전체 과정
1 사용자 입력 요청
2 요청 데이터 색인: 시스템이 사용자 요청을 색인과 연결
  * 색인은 구조화된 데이터베이스, 비정형 문서, API를 통해 얻은 프로그래밍 데이터 같은 다양한 지식 기반 정보를 포함
  * 색인의 목적: 요청과 관련된 정보를 빠르게 찾아내는 것
3 색인을 사용해 사용자 요청과 가장 관련성이 높은 데이터를 찾음
4 요청과 색인에서 검색된 관련 데이터를 결합해 강화된 입력을 생성
5 LLM에 전달 / 응답 생성
6 응답 전달

RAG에서 검색 부분의 파이프라인을 기술적으로 구현하는 과정
1 데이터 연결 / 적재: 데이터를 구분하지 않고 일단 적재
2 변환: 데이터를 분석하기 적합한 형태로 변환
3 임베딩: 데이터를 벡터 형태의 단어 임베딩으로 변환
4 저장: 벡터 데이터를 메모리나 파일 시스템 형태의 저장 시스템에 저장함 (일반적으로는 벡터 데이터베이스)
5 검색(Retreive): 저장 시스템에서 데이터를 검색하여 다음 단계에서 사용할 수 있게 함
